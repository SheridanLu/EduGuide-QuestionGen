## 第3页：什么是EduGuide？

**标题：** EduGuide：智能出题系统

**核心定义：**
> 一个基于AI的系统，能够从教学材料自动生成教育题目，采用苏格拉底式引导教学方法，并通过AI实时验证学生答案。

**核心价值：**
- 📚 **面向教师**：节省出题时间，数小时 → 数秒
- 🎯 **面向学生**：通过引导式发现学习，而非直接给答案
- 🌍 **面向所有人**：多语言支持（英语/简体中文/繁体中文）

**关键特性：**
- 自动题目生成（分层难度）
- 苏格拉底式分步引导
- AI答案验证 + 错因分析
- 交互式练习（无限次重试）
- 多文件同时上传
- Agent监控可视化
- 多语言界面

## 第4页：教育领域的挑战

**标题：** 挑战：创建优质题目很困难

**教师面临的挑战：**
- ⏰ **耗时**：创建有深度的题目需要数小时
  - 教师平均每周花费3-5小时出题
- 🎯 **难度分层**：难以创建不同难度的题目
  - 基础、中级、高级题目需要不同的方法
- 📝 **个性化**：难以提供个性化指导
  - 班级25-30名学生，每人需求不同
- 🔄 **重复性**：每次新材料都要重复相同过程

**学生面临的挑战：**
- 🤔 **直接答案不促进学习**
  - 死记硬背 vs. 深度理解
- 📉 **缺乏分步指导**
  - 卡住时需要提示，而非答案
- ❌ **犯错后无反馈**
  - 不知道为什么错，不知道如何纠正

## 第7页：系统架构

**标题：** 多Agent系统架构

**流程图：**

```
┌─────────────────────────────────┐
│   教学材料输入                    │
│   (文本/PDF/DOCX/PPTX, 支持多文件) │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  📚 Agent 1: 知识提取             │
│  • 提取关键概念                    │
│  • 输出: knowledge.json           │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  📝 Agent 2: 题目生成             │
│  • 生成分层题目(3个级别)           │
│  • 输出: questions.json           │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  🎯 Agent 3: 答案引导             │
│  • 创建苏格拉底式分步引导          │
│  • 输出: answers.json             │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  🤝 Agent 4: 补救支持             │
│  • 针对学生错误生成补救             │
│  • 输出: remedial.json            │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  🎓 交互式练习界面                 │
│  • AI验证答案                      │
│  • 错因分析 + 引导提示             │
│  • 无限次重试                      │
└─────────────────────────────────┘
```

**Agent之间通过JSON文件通信，可独立监控每个Agent的状态。**

## 第8页：多Agent设计详解

**标题：** 为什么采用多Agent架构？

| Agent | 职责 | 输入 | 输出 | 时间 |
|-------|------|------|------|------|
| **知识Agent** | 提取关键概念 | 原始文本 | 知识点列表 | ~3s |
| **题目Agent** | 生成分层题目 | 知识点 | 3级题目 | ~4s |
| **答案Agent** | 创建苏格拉底引导 | 题目 | 分步引导 | ~6s |
| **补救Agent** | 解决学生错误 | 错误+上下文 | 补救题目 | ~3s |

**设计优势：**
- ✅ **模块化**：独立更新各个Agent
- ✅ **可调试**：Agent Monitor实时监控每个Agent
- ✅ **可扩展**：为新功能添加新Agent
- ✅ **可维护**：每个Agent只做一件事

## 第10页：苏格拉底式教学

**标题：** EduGuide中的苏格拉底方法

**传统教学 vs. EduGuide：**

| 传统方式 | EduGuide |
|----------|----------|
| "NLP代表自然语言处理" | "你认为N代表什么？" |
| 直接给出答案 | 引导式发现 |
| 答错了就告诉正确答案 | 分析错因 + 给出引导 |
| 最多尝试1-2次 | 无限次重试，直到理解 |
| 被动学习 | 主动参与 |

**EduGuide的教学流程：**

```
步骤1：提出引导问题 → 学生回答
         ↓
步骤2：AI验证答案
         ↓
    ┌────┴────┐
    │ 正确？  │
    └────┬────┘
     是 ↓      ↓ 否
   下一步   🔍 分析错因
             💡 给出引导提示
             🔄 重新作答
```

**为什么重要：**
- 🧠 促进深度学习而非死记硬背
- 📊 研究表明记忆保持率提高55%
- 👨‍🏫 模仿人类教师辅导方法

## 第11页：AI答案验证（核心创新）

**标题：** AI答案验证 + 错因分析

**这是EduGuide区别于所有竞品的核心功能。**

**工作原理：**

```
学生提交答案
    ↓
调用LLM进行验证（与当前使用的AI提供商相同）
    ↓
LLM返回JSON:
{
  "correct": true/false,
  "feedback": "简要反馈",
  "error_analysis": "为什么答错 — 分析学生的误解",
  "guidance": "引导提示 — 帮助学生思考正确方向"
}
    ↓
┌─────────────┬─────────────┐
│  正确 ✅     │  错误 ❌     │
│  进入下一步   │  显示错因分析 │
│             │  显示引导提示 │
│             │  无限次重试   │
└─────────────┴─────────────┘
```

**错因分析示例：**

> ❌ 学生答案："语言模型是翻译工具"
>
> 🔍 **错因分析**：学生将语言模型的用途局限在了翻译领域。语言模型的核心功能是基于概率预测文本序列，翻译只是其众多应用之一。
>
> 💡 **引导提示**：想想语言模型在日常生活中还有哪些应用？比如手机输入法预测下一个词，这和翻译有关吗？

**与竞品对比：**
- ChatGPT：直接告诉答案 → ❌ 无验证
- Quizlet：只判断对错 → ❌ 无错因分析
- **EduGuide**：验证 + 分析 + 引导 → ✅ 完整闭环

## 第23页：致谢与问答

**标题：** 谢谢！❓

**问答准备：**

**Q1: 为什么用多Agent而不是单一AI？**
> A: 专业化。每个Agent针对其任务优化，易维护、易扩展、易调试。Agent Monitor可以实时监控每个Agent。

**Q2: AI答案验证的准确率如何？**
> A: 使用与题目生成相同的LLM，通过精心设计的prompt确保验证质量。如果验证失败，会用关键词匹配作为fallback。

**Q3: 错因分析是怎么实现的？**
> A: 当学生答错时，系统将题目、引导步骤、学生答案和期望关键点一起发送给LLM，要求返回JSON格式的错因分析和引导提示。

**Q4: 如果LLM给出错误引导怎么办？**
> A: 教师可以审查生成内容，系统是辅助工具而非替代品。多提供商支持也可以交叉验证。

**Q5: 能否支持更多语言？**
> A: 可以，多语言架构已设计好，只需添加翻译即可支持更多语言。
